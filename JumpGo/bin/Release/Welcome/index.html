New Features

Features of this version of JumpGo include a new navigation panel, a menu panel, a Settings window and more.

The most obvious feature when you start up JumpGo is going to be the new navigation panel. This new navigation panel breathes life into the old navigation buttons bringing them into 2015 with a new sleek ui. This panel contains the standard navigation buttons you're used to seeing in the beta versions of JumpGo but also holds a new menu button. This menu button opens up the brand new menu panel.

The menu panel is a feature that has popped up here and there in browsers such as Chrome, Firefox, Internet Explorer and many more, so we only thought it fitting to include it in this release of JumpGo. From the menu you now have access to parts of the browser that weren't available before the menu panel that include "New Window", Settings", Theme Designer" and "Open HTML". The menu also brings back a few old features from the beta v.1.x series and beta v.2.x series such as "New Tab", "About" and Exit".

Another feature is the Settings window. This window allows the user to change different aspects of the browser such as the version logo. JumpGo now supports switching to legacy logos from previous versions. You can also change the default search engine, the home page and the new tab page. Theme settings are available from the Settings window but for advanced theme settings it is recommened to use the Theme Designer. Addons are not currently available but will be available in the future.

One of the most recent features is the inclusion of themes. You now have more control over your customization of JumpGo. To create your own theme you can simply open the theme designer from either the menu or the Settings window and applying your theme. For creating a sharable theme, you can use a theme template downloaded from this site as a starting point and simply replace the image files with your own. (Remember to use the exact naming conventions used in the theme template.

Faster with Gecko

By using the GeckoFX renderinng engine by Mozilla, the speed of the browser was trippled from the beta v.2.x series that used the Trident rendering engine. This switch from Trident to Gecko trippled the browseing speed and reduced the number of crashes to almost none. Gecko also allows for a better html5 experience.
